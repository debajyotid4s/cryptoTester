PROJECT OVERVIEW
----------------
CryptoAlgo Tester (now "Cryptographic Realm") is a web-based cryptography 
playground designed to help users understand and experiment with classical 
encryption algorithms. It features a React/Vite frontend with Tailwind CSS 
and a Node.js/Express backend.

================================================================================
BACKEND API ENDPOINTS
================================================================================

All endpoints follow: POST /api/{algorithm}/{operation}

Health Check:
- GET /api/health

Caesar Cipher:
- POST /api/caesar/encrypt
- POST /api/caesar/decrypt

Hill Cipher:
- POST /api/hill/encrypt
- POST /api/hill/decrypt

Playfair Cipher:
- POST /api/playfair/encrypt
- POST /api/playfair/decrypt

Vigenère Cipher:
- POST /api/vigenere/encrypt
- POST /api/vigenere/decrypt

Request Format:
{
  "text": "HELLO",
  "params": {
    "shift": 3,
    "key": "KEY",
    "matrix": [[3,3],[2,5]],
    "padChar": "X"
  }
}

Response Format (Caesar - Character-based):
{
  "result": "KHOOR",
  "explain": {
    "summary": ["Sanitized input: \"HELLO\"", "Shift: 3"],
    "facts": {"m": 26, "shift": 3},
    "rows": [
      {"char": "H", "inputNum": 7, "shift": 3, "outputNum": 10, "outputChar": "K"},
      {"char": "E", "inputNum": 4, "shift": 3, "outputNum": 7, "outputChar": "H"}
    ],
    "notes": []
  }
}

Response Format (Playfair - Digraph-based):
{
  "result": "ECSPGS",
  "explain": {
    "summary": ["Sanitized input: \"HELLO\"", "Key: \"CIPHER\"", "Pairs processed: 3"],
    "facts": {"key": "CIPHER", "matrix": "[[C,I,P,H,E],...]", "pairCount": 3},
    "rows": [
      {"pair": "HE", "input": "HE", "output": "EC", "result": "EC"},
      {"pair": "LX", "input": "LX", "output": "SP", "result": "SP"},
      {"pair": "LO", "input": "LO", "output": "GS", "result": "GS"}
    ],
    "notes": []
  }
}

Response Format (Hill 2×2 - Block-based):
{
  "result": "HIOZHN",
  "explain": {
    "summary": ["Sanitized input: \"HELLO\"", "Padded with 1 \"X\""],
    "facts": {"keyMatrix": "[[3,3],[2,5]]", "det": 9, "detMod26": 9, "gcdDet26": 1},
    "rows": [
      {"block": "HE", "inputNums": [7,4], "outputNums": [7,8], "outputBlock": "HI"},
      {"block": "LL", "inputNums": [11,11], "outputNums": [14,25], "outputBlock": "OZ"},
      {"block": "OX", "inputNums": [14,23], "outputNums": [7,13], "outputBlock": "HN"}
    ],
    "padAdded": 1
  }
}

================================================================================
TECHNICAL NOTES
================================================================================

Algorithm Implementation:
- All ciphers use A=0 to Z=25 (not A=1) for mathematical consistency
- Modular arithmetic uses custom mod(n, m) to properly handle negative numbers
- Caesar: char → inputNum ± shift mod 26 → outputChar (shift default: 3)
- Vigenère: Repeating key stream aligned to plaintext, each char shifted by key char
- Playfair: 5×5 matrix from key (J→I replacement), digraph encryption with row/col/rect rules
- Hill 2×2: Block encryption via matrix multiplication, invertibility check (gcd(det,26)=1)

Text Processing:
- Text sanitization removes all non-A-Z characters and converts to uppercase
- J → I replacement (Playfair cipher requirement)
- Duplicate letters in Playfair: inserted X between duplicates (HELLO → HE-LL-OX)
- Odd-length text in Playfair: padded with X at end (HELLO → HE-LX-LO)
- Hill cipher: padded to 2-char blocks with configurable padding char (default X)

API Communication:
- Frontend proxies /api/* to http://localhost:3001 (via Vite dev server)
- Environment variable support: VITE_API_URL for custom backend URL
- Backend uses dotenv to load PORT from .env file (default 3001)
- CORS enabled for localhost:5173 and localhost:3000
- Rate limiting: 100 requests per 15 minutes per IP

Map Design:
- SVG viewBox: 680×420 (exact reference dimensions)
- 4 territories: Caesar (top-left), Vigenere (top-right), Playfair (bottom-left), Hill (bottom-right)
- Perfect tessellation: 100% coverage with zero gaps
- Hatch patterns: 8px width, 0.5 stroke width, 0.3 opacity for visual depth
- Border styling: rgba(200,190,160,0.45) for subtle territory separation

   npm start      # Runs on http://localhost:3001 (PORT from .env)

Both Servers (Full Stack):
